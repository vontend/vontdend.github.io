Prototypal inheritance

Kể từ ES6, class được giới thiệu. Javascript không có một cách định nghĩa việc kế thừa không giống như những ngôn ngữ khác, nó độc đáo ở chỗ sử dụng kế thừa nguyên mẫu (prototypal inheritance) trong khi hầu hết các ngôn ngữ khác sử dụng class-based inheritance. Điều này nghĩa là sao? Tất cả các đối tượng Javascript đều có chung một thuộc tính gọi là prototype và nó trỏ đến một đối tượng khác, đối tượng này là object prototype, đối tượng chính của chúng ta sử dụng object prototype để kế thừa thuộc tính và phương thức. Ví dụ bạn tạo một đối tượng sử dụng cú pháp sau:

const car = {}

hoặc

const car = new Object()

trong bất kỳ trường hợp nào thì thuộc tính của car là Object:

Nếu bạn khởi tạo một mảng, nó là một đối tượng:

const list = []
//hoặc
const list = new Array()

thì prototype là Array

bạn có thể kiểm tra điều này bằng Object.getPrototypeOf() và the Object.prototype.isPrototypeOf() methods:

const car = {}
const list = []

Object.getPrototypeOf(car) === Object.prototype
Object.prototype.isPrototypeOf(car)

Object.getPrototypeOf(list) === Array.prototype
Array.prototype.isPrototypeOf(list)

Tất cả các thuộc tính và phương thức của prototype thì luôn sẵn có với đối tượng sở hữu prototype đó:

Object.prototype là nguyên mẫu chung cho tất cả các đối tượng

Object.getPrototypeOf(Array.prototype) == Object.prototype

Bạn sẽ tự hỏi prototype của Object.prototype là gì, là null, nó là đặc biệt.

Ví dụ trên bạn có thể thấy cách mã chuỗi prototype làm việc. Bạn có thể tạo một Object mà nó lại kế thừa Array và nếu như bạn tạo thêm bất kỳ đối tượng nào thừa kế nó sẽ có cả Array và Object prototype, đồng thời kế thừa tất cả thuộc tính và phương thức của cả Array và Object.

Ngoài việc sử dụng toán tử new để khởi tạo một đối tượng hoặc sử dụng cú pháp literal cho cả đối tượng và mảng, bạn có thể khởi tạo bằng cách dùng Object.create()

Tham số đầu tiên được xem như kiểu prototype

const car = Object.create({})
const list = Object.create(Array)

chú ý bởi vì bạn có thể khởi tạo một Array bằng cách:

const list = Object.create(Array.prototype)

và trong trường hợp này Array.isPrototypeOf(list) sẽ bằng false, trong khi Array.prototype.isPrototypeOf(list) sẽ bằng true.

Định nghĩa một class

class Person {
  constructor(name) {
    this.name = name
  }
  hello() {
    return 'Hello, I am ' + this.name + '.'
  }
}

Một class có một định danh để bạn có thể sử dụng khởi tạo đổi tượng bằng cách sử dụng new ClassIndentifier()

Khi một class được khởi tạo thì constructor sẽ được gọi với tất cả các tham số được truyền vào.

Một class cũng có thể có bao nhiêu phương thức tùy vào nhu cầu của bạn. Trong trường hợp này hello là một phương thức và có thể được gọi trên tất cả các đối tượng khởi tạo bằng class này:

const flavio = new Person ('Flavio')
flavio.hello()

Kế thừa

Một class có thể kế thừa một class khác, và đối tượng được khởi tạo từ class kế thừa sẽ có tất cả các phương thức của cả 2 class.

Nếu class kế thừa có một phương thức trùng tên với class cao hơn thì phương thức gần nhất sẽ được ưu tiên:

class Programmer extends Person {
  hello() {
    return super.hello() + ' I am a programmer.'
  }
}
const flavio = new Programmer('Flavio')
flavio.hello() // Hello I'm Flavio, I am a programer

Các lớp không có khai báo biến rõ ràng nhưng bạn có thể khởi tạo bất kỳ biến nào bên trong constructor.

Trong một class bạn có thể tham chiều đến class cha bằng cách gọi super().

Phương thức static

Thông thường các phương thức được định nghĩa trên instance chứ không phải trên một class.

Thay vào đó thì các phương thức static được thực thi trên class:

class Person {
  static genericHello() {
    return 'Hello'
  }
}
Person.genericHello() //Hello

Phương thức private

Javascript không tích hợp sẵn cách xác định phương thức private hoặc protected.

Sẽ có cách giải quyết nhưng chúng ta tạm thời không tìm hiểu.

Setter và getter

Bạn có thể thêm tiền tố set hoặc get trước tên phương thức để tạo setter và getter, hai phần mã này thực thi khác nhau tùy thuộc vào bạn đang muốn làm gì, truy cập giá trị của biến hay sửa giá trị của nó.

class Person {
  constructor(name) {
    this.name = name
  }
  set name(value) {
    this.name = value
  }
  get name() {
    return this.name
  }
}

Nếu bạn chỉ có getter thì thuộc tính không thể sửa đổi, bất kỳ cố gắng sửa đổi nào sẽ bị bỏ qua.

class Person {
  constructor(name) {
    this.name = name
  }
  get name() {
    return this.name
  }
}

Nếu bạn chỉ có setter, bạn có thể thay đổi giá trị nhưng lại không thể truy cập bên ngoài class.

class Person {
  constructor(name) {
    this.name = name
  }
  set name(value) {
    this.name = value
  }
}
