Module

ES Module là chuẩn của ECMAScript để làm việc với các module.

Trong khi Node.js sử dụng chuẩn CommonJS trong nhiều năm, các trình duyệt chưa bao giờ có được một hệ thống module, vì mọi quyết định như là hệ thống module phải được chuẩn hóa trước bởi ECMAScript trước khi các trình duyệt thực hiện tích hợp.

Module tuyệt vời ở chỗ nó cho phép bạn gói gọn và sắp xếp các chức năng trong một file javascript khác, dạng như thư viện. Nếu làm việc với composer của PHP có lẽ bạn đã biết điều này từ lâu rồi.

Cú pháp Module

import package from 'module-name'

trong khi chuẩn CommonJS sử dụng:

const package = require('module-name')

Một module là một file Javascript mà nó xuất (export) một hay nhiều giá trị khác nhau (object, hàm, biến) sử dụng từ khóa export. Ví dụ trong module sau đây export một hàm trả một string về chữ hoa:

uppercase.js

export default str => str.toUpperCase()

Trong ví dụ này, module được định nghĩa theo cách mà mình xin gọi là module đơn, do đó nó có thể là một hàm anonymous (không cần tên). Ngược lại, nó cần một tên để phân biệt với các phần export khác của module.

Bây giờ bất kỳ module Javascript nào khác đều có thể sử dụng chức năng của uppercase.js bằng cách import nó.

Một file HTML cũng có thể import một module bằng cách:

<script type="module" src="index.js"></script>

Chú ý: phương pháp import module này giống như bạn load script bằng cách defer.

Một điểm cần lưu ý khác là bất kỳ script nào được load với type="module" đều bật strict mode.

Trong ví dụ này, uppercase.js được định nghĩa default export, vì vậy khi chúng ta import nó, chúng ta được quyền gán cho nó một tên bất kỳ.

import toUpperCase from './uppercase.js'

và dùng chúng:

toUpperCase('test') //TEST

Bạn cũng có thể import một đường dẫn tuyệt đối để tham chiếu một module trên một domain khác.

import toUpperCase from 'https://flavio-es-modules-example.glitch.me/uppercase.js'

Cú pháp sau đây đúng:

import { foo } from '/uppercase.js'
import { foo } from '../uppercase.js'

Và sai:

import { foo } from 'uppercase.js'
import { foo } from 'utils/uppercase.js'

Chỉ có thể là tuyệt đối hoặc bắt đầu với ./ \/ trước tên file.

Những cách import, export khác

Hãy nhìn lại ví dụ phía trên:

export default str => str.toUpperCase()

Module này tạo một default export. Tuy nhiên bạn có thể export nhiều nhứ hơn nữa:

const a = 1
const b = 2
const c = 3
export { a, b, c }

Khi import có thể import tất cả sử dụng:

import * from 'module'

Hoặc chỉ vài giá trị sử dụng cấu trúc destructing:

import { a } from 'module'
import { a, b } from 'module'

Bạn có quyền thay đổi tên biến bằng cách sử dụng từ khóa as

import { a, b as two } from 'module'

Bạn có thể import default, không default trong cùng một cú pháp như các file React component thường bắt đầu:

import React, { Component } from 'react'

CORS

Module được import từ một domain khác sử dụng CORS, chúng phải có một CORS header đúng đắn để cho phép cross-site loading (như Access-Control-Allow-Origin: *)

Nếu trình duyệt không hỗ trợ:

Sử dụng sự kết hợp của type="module" và nomodule:

<script type="module" src="module.js"></script>
<script nomodule src="fallback.js"></script>

ES Module là một phần tính năng lớn nhất trong ES6 được giới thiệu đến các trình duyệt hiện đại. Nó là một phần của ES6 nhưng con đường thực hiện nó là rất dài.

Chúng ta hiện có thể sử dụng nó. Nhưng cần nhớ rằng có hơn một vài module sẽ lập tức gây ra vấn đề về hiệu năng đối với ứng dụng của chúng ta bởi vì trình duyệt cần làm thêm một bước trước khi thực hiện chạy chương trình.

Webpack là một lựa chọn an toàn ngay cả khi trình duyệt có tích hợp module. Các sản phẩm sau này bạn làm với React hay React Native thì webpack là một điều không thể thiếu. Nói một cách đơn giản Webpack là module loader đóng gói tất cả module vào một vài tập tin Chúng ta sẽ còn gặp lại webpack trên website này.
