Cài đặt

Làm sao để cài đặt React?

React chỉ là một thư viện, nói cài đặt nghe có vẻ hơi lạ. Dùng từ thiết lập có lẽ sẽ đúng hơn nhưng tôi thích dùng từ cài đặt hơn.

Có rất nhiều cách để cài đặt React để có thể sửng dụng có ứng dụng hoặc website của bạn.

Nhúng trực tiếp vào website của bạn

Cách nhanh nhất là thêm một file Javascript trực tiếp vào website của bạn. Điều này sẽ OK trong trường hợp ứng dụng React của bạn sẽ tương tác với các thành phần giao diện mà không thực sự kiểm soát toàn bộ khía cạnh điều hướng.

Trong trường hợp này, bạn thêm vào 2 thẻ script ở cuối thẻ body:

<html>
  ...
  <body>
    ...
    <script
      src="https://cdnjs.cloudflare.com/ajax/libs/react/16.8.3/umd/react.development.js"
      crossorigin
    ></script>
    <script
      src="https://cdnjs.cloudflare.com/ajax/libs/react-dom/16.8.3/umd/react-dom.production.min.js"
      crossorigin
    ></script>
  </body>
</html>

Hãy dùng version mới nhất của React tại thời điểm bạn đọc bài viết này.

Ở đây chúng ta đã tải cả React và React DOM. Tại sao lại là hai thư viện ? Bởi vì React bản thân nó 100% độc lập với trình duyệt và có thể sử dụng riêng bên ngoài trình duyệt ví dụ như các thiết bị di động với React Native. Do đó chúng ta cần React DOM để thêm trình bao bọc (wrapper) cho trình duyệt.

Sau các thẻ trên bạn có thể nhúng các file Javascript sử dụng React hoặc thậm chí là sử dụng inline.

<script src="app.js"></script>
<!-- or -->
<script>
  //my app
</script>

Để sử dụng JSX bạn cần thêm một bước nữa: Babel

<script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>

Tiếp theo là sử dụng các file script của bạn với MIME type là text/babel:

<script src="app.js" type="text/babel"></script>

Và bây giờ bạn có thể sử dụng JSX trong file app.js của mình:

const Button = () => {
  return <button>Click me!</button>
}
ReactDOM.render(<Button />, document.getElementById('root'))

Hãy thử ví dụ sau đây tại Glitch:

https://glitch.com/edit/#!/react-example-inline-jsx?path=script.js

Bắt đầu theo cách này có vẻ sẽ nhanh nhưng hầu hết chả ai dùng cách này, nó chỉ dùng để test bậy bạ thì được.

Sử dụng create-react-app

create-react-app là một dự án nhằm giúp bạn nhanh chóng cài đặt React mà gần như không mất một chút thời gian nào. Bất kỳ một ứng dụng React nào đều cần một cấu trúc như vậy và create-react-app đáp ứng được điều đó.

Đầu tiên bạn cần cài đặt Node.js tại website https://nodejs.org, hãy làm theo các hướng dẫn trên website của họ. Cái chúng ta cần ở đây thực chất không phải node mà là npm được tích hợp sẵn trong Node.

Hãy chắc chắc rằng npm đã được cài đặt trong máy của bạn bằng cách vào Terminal/CMD/Gitbash gõ npm -v.

Tiếp theo bạn vào Terminal dùng lệnh npm install -g create-react-app để cài đặt vào máy mình. Với lệnh này bạn chỉ cần làm một lần và sau này mỗi khi bạn tạo dự án mới chỉ cần dùng create-react-app my-sampe-app.

Sau khi bạn chạy create-react-app my-sample-app cài đặt xong thì màn hình Terminal của bạn sẽ có dạng như sau:

create-react-app đã tạo cho bạn toàn bộ các file và thư mục cần thiết cho một ứng dụng React, có thể coi đây là một ứng dụng React.

Như hình ảnh trên, hiện tại Facebook khuyến khích bạn dùng yarn (tương đương với npm) đồng thời hãy xem qua file packages.json của bạn

{
  "name": "my-sample-app",
  "version": "0.1.0",
  "private": true,
  "dependencies": {
    "react": "^16.8.4",
    "react-dom": "^16.8.4",
    "react-scripts": "2.1.8"
  },
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test",
    "eject": "react-scripts eject"
  },
  "eslintConfig": {
    "extends": "react-app"
  },
  "browserslist": [
    ">0.2%",
    "not dead",
    "not ie <= 11",
    "not op_mini all"
  ]
}

Bây giờ bạn có thể chạy ứng dụng của bạn với lệnh yarn start hoặc npm run start.

cùng với đó create-react-app sẽ mở trình duyệt và bạn được kết quả trên màn hình tương tự như sau:

Ngoài yarn start chúng ta còn một số lệnh khác:

  • yarn build: Dùng để build toàn bộ ứng dụng React của chúng ta, được lưu vào thư mục build để bạn tiến hành host lên server cho người dùng sử dụng.
  • yarn test: Chạy test code với Jest (đây là javascript nâng cao bạn nên tìm hiểu sau và nên tìm hiểu).
  • npm eject: eject khỏi create-react-app,

eject nghĩa là một hành động mà bạn quyết định rằng create-react-app đã làm đủ các công việc cần thiết cho bạn, nhưng bạn muốn thực hiện nhiều hơn những gì nó cho phép. Bởi vì create-react-app là một tập hợp các quy ước với một số lượng tuỳ chọn giới hạn. Đến một lúc nào đó, nhu cầu của bạn sẽ đòi hỏi một cái gì đó độc đáo hơn và vượt xa khả năng của create-reat-app.

Khi bạn eject, bạn sẽ mất đi tính năng tự cập nhật nhưng bạn sẽ có được sự linh hoạt hơn trong việc cấu hình webpack, Babel.

Khi bạn eject bạn không thể làm ngược lại, không thể khôi phục ứng dụng như ban đầu, thay vào đó hãy dùng git. Sau khi eject bạn sẽ có hai thư mục mới là config và scripts. Chúng chứa cấu hình và bạn có thể bắt đầu sửa chúng.

Những cách cài đặt khác

Ngoài hai cách trên còn một số cách khác để bạn có thể sử dụng react như Code Sanbox, Codepen, mình không tiện nêu, bạn có thể tự tìm hiểu, bản thân chúng chỉ có thể tiện cho việc test các ứng dụng mà thôi.
